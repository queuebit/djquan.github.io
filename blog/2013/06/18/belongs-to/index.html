Belongs To

Today was spent mostly working on my actual bake sale event class. It definitely took longer than I had expected, and I’m only mostly done.

Russ Olsen, at RubyNation, was talking about how programming and web development is a profession where progress is not made linearly. It’s based on problem solving and “A-ha!” moments. To paraphrase: A lot of the time, it’s not you work for X hours and you get Y done.

Some issues I ran into today include dealing with strong parameters for a model that “belongs to” a devise model and dealing with form helpers for nested attributes. Up till now, form helpers just worked and I didn’t really need to understand how they worked (a lot of rails can be like that). But, when you deviate from normal convention a little, you really begin to understand them better as you work through issues you are having.

Another mention for code-climate. Having it follow an open source github repo is easy. It’ll then create a site that you can refresh at will. The statistics are nice, and I just noticed that they were pointing to a code smell of mine: the dreaded code duplication. The link is here, but hopefully I’ll have time to fix it soon. My duplication is that my organization and user model check the same validation on usernames. Not horrible, but I’m sure there’s a way to extract it to a helper.

Check out Rails-ERD(Entity-Relationship Diagrams). It’s really easy to install. Basically, it automatically creates diagrams based on the models you have and their relationships. It’s a nice way to see relationships.

So right now it’s my app is fairly simple (and look how much stuff devise adds..), but it should start to have some more advanced logic soon.

Tomorrow will be a somewhat lighter day; I’ll finish up the event class. So that’ll include the date/time logic and forms for inputing that. I might also look into Spring to speed up tests.

I also started the Startup Engineering class by Stanford at Coursea. So far, it’s pretty interesting – it does focus on javascript skills which is a nice benefit. It’s good to learn multiple frameworks and languages.

Also, thanks queuebit for the pull request. I had a broken link, and queubit graciously sent a pull request to fix it. A nice use of a PR as my entire blog site, including content, is on github.

Random vim newbie lesson. :set spell (from the command mode, only when you need it)

I do all my blog writing in vim. When I’m somewhat finished writing, I’ll enter the above command and see spelling errors underlined. Definitely not the most revolutionary idea, but it’s worth a mention. I don’t use it enough to warrant make and memorize a key binding though.
